01 ├втВмтАЭ What Is FunBot?

CORE

06 ├втВмтАЭ NATT Ghost Agent (/natt)

GHOST

├░┼╕тАШ┬╗ What Is NATT?

NATT = Network Attack & Testing Toolkit. It's a ghost-mode ethical hacker agent that infiltrates targets with minimal footprint and produces detailed security intelligence.

├░┼╕тАЬ┬б Command Syntax

/natt <target> [--mode=passive|stealth|active] [--mission=mission_type] [--auth=proof_string] [--type=url|ip|domain|api-endpoint]

├░┼╕┼╜┬п Target Types

├░┼╕┼Т┬Р https://example.com ├втАатАЩ auto-detected as URL ├░┼╕тАЭ┬в 192.168.1.1 ├втАатАЩ auto-detected as IP ├░┼╕тАЭ┬д api.example.com ├втАатАЩ auto-detected as domain ├░┼╕тАЭтАФ https://api.x.com/v1/users ├втАатАЩ API endpoint

├в┼б┬б Ghost Modes

passive Observation only. No traffic sent to target. Reads public data, DNS, WHOIS. Safest.
stealth Light probing. Minimal footprint. Avoids triggering alerts. Default mode.
active Full exploitation simulation. Requires --auth= proof. Requires authorization.

├░┼╕┼╜┬п Mission Types

MissionDescription
web-appWeb application security (XSS, SQLI, CSRF...)
html-analysisPage structure, forms, hidden fields
api-reconAPI endpoint discovery & auth testing
network-reconPort scan, banner grabbing, topology
osintOpen-source intelligence gathering
auth-testingLogin, session, JWT, OAuth flows
full-ghostAll of the above in one mission (default)

├░┼╕тАЬ┬Э Examples

# Simple recon (stealth, full-ghost ├втВмтАЭ defaults) /natt https://example.com # API recon in stealth /natt https://api.myapp.com/v1 --mission=api-recon # Passive OSINT only /natt targetdomain.com --mode=passive --mission=osint # Full active test (requires authorization proof) /natt https://myapp.com --mode=active --auth=pentest-auth-2026 --mission=full-ghost # IP / network scan /natt 192.168.1.0/24 --mission=network-recon

Step-by-Step: Run Your First NATT Mission

  1. Confirm you have authorization
    You must own the target or have written permission. NATT checks this ├втВмтАЭ no auth = no mission.
  2. Choose your mode
    Start with passive or stealth (default). Only use active with explicit --auth= proof.
  3. Pick a mission type (optional)
    Leave blank for full-ghost. Use a specific type like api-recon to focus the scan.
  4. Run the command
    /natt https://yourapp.com --mode=stealth --mission=web-app
  5. Review the findings in Slack
    FunBot posts a structured report with risk rating, findings, and recommendations.
  6. Generate a PowerPoint report
    Use /natt-report to get a full slide deck of your missions over any date range.

08 ├втВмтАЭ Cron Scheduler (/natt-cron)

CRON

├в┬П┬▒├п┬╕┬П What Is the Cron Scheduler?

Schedule NATT reports to be generated and delivered to Slack channels automatically ├втВмтАЭ hourly, daily, weekly, monthly, or on any cron expression you choose. Reports appear as PowerPoint files in the scheduled channel.

├░┼╕тАЬтА╣ All Subcommands

SubcommandAction
listShow all scheduled jobs
add <options>Create a new scheduled job
healthCheck system health & overdue jobs
run <jobId>Trigger a job immediately
pause <jobId>Pause a job (keeps its schedule)
resume <jobId>Resume a paused job
remove <jobId>Delete a job permanently

├░┼╕тАЬтАж Cadences & Default Times

CadenceDefault Schedule
hourlyEvery hour on the hour
dailyEvery day at 8:00 AM
weeklyMonday 8:00 AM
biweekly1st and 15th of month, 8:00 AM
monthly1st of month, 8:00 AM
customYou provide the cron expression

├░┼╕тАЭ┬з Add Parameters

ParameterRequiredDescription
name="..."yesDisplay name for the job
cadence=yeshourly/daily/weekly/biweekly/monthly/custom
window=yesDate window for the report (see below)
channel=yesSlack channel (e.g. #security)
cron="..."custom onlyCron expression (e.g. "0 9 * * 1,3,5")
tz=optionalTime zone (default: UTC)
operator=optionalFilter by operator Slack ID
title="..."optionalReport filename title
empty=skip|sendoptionalWhat to do if no missions found

├░┼╕тАЬтАж Window Values

WindowDate Range Used
last-24hLast 24 hours
last-7-daysLast 7 days
last-30-daysLast 30 days
last-monthFull previous calendar month
mtdMonth-to-date (1st ├втАатАЩ today)
ytdYear-to-date (Jan 1 ├втАатАЩ today)
customUse from= and to= dates

├░┼╕тАЬ┬Э cron add Examples

# Weekly report every Monday ├втВмтАЭ last 7 days ├втВмтАЭ delivered to #security /natt-cron add name="Weekly Security Report" cadence=weekly window=last-7-days channel=#security # Daily standup brief at 8 AM ├втВмтАЭ last 24 hours /natt-cron add name="Daily Standup Brief" cadence=daily window=last-24h channel=#natt-reports # Monthly executive summary ├втВмтАЭ full previous month /natt-cron add name="Monthly Executive" cadence=monthly window=last-month channel=#security-mgmt # Mon/Wed/Fri 9 AM custom cron ├втВмтАЭ month-to-date /natt-cron add name="MWF MTD Brief" cadence=custom cron="0 9 * * 1,3,5" window=mtd channel=#security # Biweekly with specific time zone /natt-cron add name="Biweekly EST" cadence=biweekly window=last-30-days channel=#security tz=America/New_York

├░┼╕тАЬ┬Э Managing Jobs

# See all jobs and queue stats /natt-cron list # Check health ├втВмтАЭ see error/overdue jobs /natt-cron health # Trigger immediately (by job ID shown in /natt-cron list) /natt-cron run cron-1740234567-abc12 # Pause without deleting /natt-cron pause cron-1740234567-abc12 # Resume a paused job /natt-cron resume cron-1740234567-abc12 # Permanently delete /natt-cron remove cron-1740234567-abc12

Step-by-Step: Schedule Your First NATT Report

  1. Decide on cadence and window
    Example: every Monday covering last 7 days.
  2. Pick the Slack channel for delivery
    The PowerPoint file will be posted here automatically.
  3. Create the job
    /natt-cron add name="Weekly Intel" cadence=weekly window=last-7-days channel=#security
  4. Verify it's scheduled
    /natt-cron list ├втВмтАЭ your job should appear with status Active.
  5. Test it right now
    Copy the job ID from the list and run /natt-cron run <jobId> to trigger immediately.
  6. Check health periodically
    /natt-cron health ├втВмтАЭ FunBot also alerts you automatically on failure.
